⚡ 3. Animation JavaScript (Drawing Effect)

Animation de "dessin progressif" utilisant stroke-dashoffset :

// Sélectionner toutes les lignes avec classe 'canalisation-line'
document.querySelectorAll('.canalisation-line').forEach(path => {
  const svgPath = path as SVGGeometryElement;
  
  // 1. Calculer la longueur totale du path
  const len = svgPath.getTotalLength();
  
  // 2. Définir dasharray et dashoffset initiaux (ligne invisible)
  path.style.strokeDasharray = `${len}px`;
  path.style.strokeDashoffset = `${len}px`;
  
  // 3. Démarrer l'animation après 500ms
  setTimeout(() => {
    path.style.transition = 'stroke-dashoffset 6s ease-in-out';
    path.style.strokeDashoffset = '0px';
  }, 500);
});

Timing :

  • Délai initial : 500ms (avant démarrage)
  • Durée : 6 secondes
  • Easing : ease-in-out (démarrage doux, fin douce)

🔄 4. Adaptation pour ArticlePlaceholder (viewBox 400×200)

Pour adapter ces paths au viewBox 400×200 de ArticlePlaceholder :

Proportions originales du portail :

  • Ligne 1 : viewBox 177×159
  • Ligne 2 : viewBox 176×59

Méthode de conversion :

  1. Identifier les points clés des paths (départ, virages, arrivée)
  2. Calculer les proportions relatives (x/177, y/159)
  3. Multiplier par les nouvelles dimensions (400×200)
  4. Reconstruire les paths avec coordonnées cubiques de Bézier pour virages arrondis

Exemple pour Ligne 1 (Haut-droite → Bas-gauche) :

// ORIGINAL (177×159) :
M176 1 L53.5359 1 C52.4313 1 51.5359 1.89543 51.5359 3 L51.5359 56 C51.5359 57.1046 50.6405 58 49.5359 58 L0 58

// ADAPTÉ (400×200) :
M 390 5 L 130 5 C 128 5 126 7 126 9 L 126 155 C 126 157 124 159 122 159 L 10 159

Points de conversion :

  • 176/177 ≈ 99% de la largeur → 0.99 × 400 ≈ 390
  • 53.5359/177 ≈ 30% de la largeur → 0.30 × 400 ≈ 130
  • 56/159 ≈ 35% de la hauteur → 0.35 × 200 ≈ 155
